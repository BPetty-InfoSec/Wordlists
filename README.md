<h1>buildlist.py</h1>
<p>This project came about because I needed to collect some wordlists for penetration testing. It is still very rough, and doesn't handle name conflicts well at all at this point.</p>
<p>Clone this repo, and it will create the Wordlists directory where you choose, with this file, buildlist.py, and the lib folder within it. That's all. From this point, simply download lists into the lib folder. I recommend using subfolders for each source, so that you can keep them separate. Also, buildlist.py will ignore anything (including directories) that begin with ".git", or are named "README.md" or "CREDIT.txt". This is so that it is easy to git clone repos containing wordlists, and create a CREDIT.txt file so that attribution is taken care of.</p>
<p>Some repos have wordlists in compressed format, and buildlist.py does NOT handle these at this point, so you will need to monitor the downloads and manually decompress any files that are located within...and then remove the archive file, so that it is not also symlinked.</p>
<p>After the different wordlists are downloaded, simply run buildlist.py from the main Wordlists directory. This will create <i>another</i> Wordlists directory inside the first one, this one containing symlinks to all of the files in the various and sundry directories within lib.</p>
